# 测试要求

## 最低测试覆盖率：80%

测试类型（全部必需）：
1. **单元测试** - 单独函数、工具、组件
2. **集成测试** - API 端点、数据库操作
3. **端到端测试** - 关键用户流程（每种语言选择框架）

## 测试驱动开发

强制工作流：
1. 先写测试（红色）
2. 运行测试 - 应该失败
3. 写最小实现（绿色）
4. 运行测试 - 应该通过
5. 重构（改进）
6. 验证覆盖率（80%+）

## 测试故障排除

1. 使用 **tdd-guide** 智能体
2. 检查测试隔离
3. 验证 mock 是否正确
4. 修复实现，而非测试（除非测试本身有误）

## 智能体支持

- **tdd-guide** - 新功能时主动使用，强制先写测试
