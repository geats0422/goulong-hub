# 通用模式

## 骨架项目

实现新功能时：
1. 搜索经过实战测试的骨架项目
2. 使用并行智能体评估选项：
   - 安全性评估
   - 可扩展性分析
   - 相关性评分
   - 实施规划
3. 克隆最佳匹配作为基础
4. 在已验证的结构中迭代

## 设计模式

### 仓库模式

在一致的接口背后封装数据访问：
- 定义标准操作：findAll、findById、create、update、delete
- 具体实现处理存储细节（数据库、API、文件等）
- 业务逻辑依赖抽象接口，而非存储机制
- 便于轻松切换数据源，并简化使用 mock 进行测试

### API 响应格式

对所有 API 响应使用统一包装：
- 包含成功/状态指示器
- 包含数据负载（错误时可为 null）
- 包含错误消息字段（成功时可为 null）
- 包含分页响应的元数据（总数、页码、限制）
