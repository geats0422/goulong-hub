# 工作流规范（基于 learn-evolve 20260627-001）

## 五阶段全流程

涉及多个子系统的新功能必须走完整流程：

```
brainstorm → grill-me → plan → execute → finish
```

- **brainstorm**：一次一个问题，逐一确认设计决策
- **grill-me**：压力测试每个决策分支，追问到叶子节点
- **plan**：拆解为 2-5 分钟的可执行任务
- **execute**：TDD 串行执行（红灯→实现→转绿→规格审查→质量审查），中间不暂停
- **finish**：最终验证→提交→PR→合并

## 关键原则

- execute 阶段中间不暂停询问"是否继续"，连续执行到所有任务结束
- 每个实现任务必须先写失败测试
- 提交粒度：一个逻辑变更一个 commit
- 从 `main` 创建 feature 分支，PR 合并后删除分支
