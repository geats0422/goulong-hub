# 上下文操作

管理跨会话的知识持久化。

## 保存上下文

当用户说 `/learn` 或会话自然结束时：

### 记录内容
1. **关键决策** — 做了什么架构/设计决策及原因
2. **剩余工作** — 哪些任务还在进行中
3. **经验教训** — 本次会话中发现的模式、需要注意的坑
4. **项目当前状态** — git 分支、未提交的变更

### 保存格式

保存到 `~/.opencode/learnings/<project>/YYYY-MM-DD-session.md`:

```markdown
# 会话记录 - YYYY-MM-DD

## 项目状态
- 分支: feature/xxx
- 关键变更: [简述]

## 关键决策
- [决策]: [原因]

## 经验教训
- [发现]: [将来如何应对]

## 剩余工作
- [ ] 待办事项
```

## 恢复上下文

在开始新会话时：
1. 读取最近的 learnings 文件
2. 了解上次会话做了什么
3. 继续未完成的工作

## 管理学习记录

定期（每周或每两周）：
1. 审查累积的 learnings
2. 将反复出现的经验提升为 rules 或 skill
3. 清理不再相关的旧记录
