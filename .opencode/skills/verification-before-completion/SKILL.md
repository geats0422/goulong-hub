# 完成前验证

禁止在未实际验证的情况下声称工作完成。

## 铁律

**不能假设代码能运行。必须运行验证命令来证明它能运行。**

## 验证清单

在声明"完成了"之前，你必须完成以下全部步骤：

1. **构建通过** — 运行项目的 build 命令，确认无错误
2. **测试通过** — 运行项目的 test 命令，确认全部通过
3. **类型检查** — 如果项目使用 TypeScript，运行 tsc --noEmit
4. **Lint 通过** — 运行项目的 lint 命令，确认无警告
5. **新功能验证** — 如果有新功能，写一个快速的验证脚本来确认核心流程能运行

## 验证流程

```
对于每个验证步骤:
  1. 运行验证命令
  2. 读取输出
  3. 确认没有错误/失败
  4. 如果失败 → 修复 → 重新验证（不要跳过）
  5. 只有在所有验证通过后才声明"完成"
```

## 反模式

- ❌ "代码看起来应该能运行" → 运行它来确认
- ❌ "编译通过就是完成了" → 测试也必须要跑
- ❌ "我只是改了注释，不需要验证" → 仍然需要跑验证以确认
- ❌ "验证命令运行失败但看起来是环境问题" → 修复环境问题，不要凭空假设

## 输出格式

```markdown
## 验证结果

| 检查项 | 命令 | 结果 |
|--------|------|------|
| Build  | npm run build | ✅ 通过 |
| Test   | npm test      | ✅ 42 passed |
| Type   | tsc --noEmit  | ✅ 无错误 |
| Lint   | npm run lint  | ✅ 无警告 |

所有验证通过，可以声明完成。
```
