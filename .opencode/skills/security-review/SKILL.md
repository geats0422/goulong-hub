# 安全审查清单

在提交任何代码之前检查以下安全要点。

## 必须检查的项目

### 密钥管理
- [ ] 没有硬编码的 API key、密码、token 或私钥
- [ ] 密钥通过环境变量或密钥管理服务获取
- [ ] .env 文件在 .gitignore 中
- [ ] 没有将密钥打印到日志或控制台

### 输入验证
- [ ] 所有用户输入在系统边界经过验证
- [ ] 文件上传有类型检查（MIME + 扩展名）
- [ ] 文件上传有大小限制
- [ ] URL 参数经过验证和清理
- [ ] 验证失败返回清晰的错误消息（不泄露内部实现）

### 注入防护
- [ ] SQL 查询使用参数化查询，绝不拼接字符串
- [ ] HTML 输出经过转义防止 XSS（或使用安全的模板引擎）
- [ ] 命令执行使用参数数组，不使用 shell 字符串拼接

### 认证与授权
- [ ] 所有需要认证的端点验证了用户身份
- [ ] 没有越权访问的可能（用户 A 不能访问用户 B 的数据）
- [ ] Token 有过期时间
- [ ] 敏感操作需要重新认证

### 数据传输
- [ ] HTTPS 用于所有数据传输
- [ ] Cookie 设置了 HttpOnly、Secure、SameSite 属性
- [ ] CORS 配置不包含 `*` 通配符

### 错误处理
- [ ] 错误消息不泄露内部实现细节
- [ ] 不在客户端返回堆栈跟踪
- [ ] 数据库错误被捕获并转换为通用错误消息

## 问题处理

如果发现任何问题：

1. **CRITICAL** (密钥泄露、未授权访问) → STOP → 立即通知 → 修复后轮换密钥
2. **HIGH** (注入漏洞、认证绕过) → 优先修复 → 完成后审查同类问题
3. **MEDIUM** (配置不安全、信息泄露) → 记录追踪 → 在近期修复
4. **LOW** (深度防御建议) → 记录 → 可后续处理
